RailScape: A Unified Simulation Framework for Passenger and Trackside Perception in Intelligent Railway Systems

Generation of diverse scenarios across trackside, platform, and vehicle interior, in different environments and vehicles, simulated in Isaac Sim.

Abstract

Autonomous train operation (ATO) for on-sight driving, particularly for passenger transport in open environments, demands perception systems that robustly interpret the environment around and inside the vehicle. Since real-world data collection and system validation are costly and hard to scale, simulation is essential. Existing railway simulators, however, do not address the passenger-exchange domain, where the vehicle interacts with passengers at the platform and inside the vehicle. We address this gap with RailScape, a simulation framework covering the vehicle exterior and interior across the complete operational cycle. A dedicated railway domain model captures semantics of tracks, platforms, and vehicle interiors. A probabilistic Python API specifies the dynamic behavior of passengers and vehicles through randomized scenario parameters and provides an extensible toolbox of primitive behavior. An Isaac Sim backend simulates sensor data with human animation, integrating existing digital models of vehicles and stations through a generic interface. The framework supports applications from synthetic data generation to closed-loop scenario-based testing of ATO systems. In a case study, we use it to evaluate a camera-based passenger counting system during boarding under varying crowd density.
